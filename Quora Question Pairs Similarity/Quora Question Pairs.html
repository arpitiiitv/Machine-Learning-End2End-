Quora Question Pairs Similarity

1. Problem and Goal¶

1.1 What is Quora:¶

Quora is very popular plateform for learners, here we can ask question and get answers of any kind of topics. It is very usefull for knowledge purpose for students, professionals, teachers and industry leaders all keen to share their knowledge. Quora has approximatly 300 million monthly active users of all field, Moslty everyone visits quora if they are looking for something on internet. It is one of the best social site for learners.

1.2 Problem statement¶

We asks questions on quora, two or more people may ask the same questions in different wordings, The meaning of question is same but they had used differnt sentences. For example :

  • Q1: Where can I get cheap food in Kanpur?
  • Q2 Where are the cheap food sold in Kanpur?
  • Q3 How can I get cheap food in kanpur?
All the questions have same meaning but they are worded differntly. So our goal is to find duplicate questions and merge their answers so that we can get better results.
  • Find and merge the questions which are already asked on quora.
  • If someone asks any question, we should give them answers if that question is already answered in past.
  • Aim of this project is to predict if Question1 is duplicate of Question2.

1.3 Useful Links for this project¶

Dataset is taken from https://www.kaggle.com/c/quora-question-pairs

2. Engineering problem¶

2.1 Know the dataset¶

dataset is taken from https://www.kaggle.com/c/quora-question-pairs

Data will be in a file Train.csv
Train.csv contains 6 columns : id, qid1, qid2, question1, question2, is_duplicate
Size of Train.csv - 60MB
Number of rows in Train.csv = 404,290

2.2 Problem specification¶

It is a binary classification task. We have 2 possible output for is_duplicate (Yes/No).

False Negative Rate

Model predicted negative and it was false.
Our model predicted two questions are not duplicate but they are duplicate.

False Positive Rate

Model predicted positive and it was false.
Our model predicted two questions are duplicate but they are different.

High False positive rate will affect model badly, if we will merge the answer of two different questions it we be very bad and customers will not visit this site again. We have to minimize FPR for this problem.

2.3 Problem Constraint¶

  • Cost of misclassification is very high.
  • We will use Log-loss as primary performance metric
  • Confusion matrix will be secondary performance metric to check FPR.

Feature Engineering

4.1 Creating artificial Features¶

From domain knowledge, we can create many new features from the given dataset that may help the model to perform better.

Observations¶

     
  • If two questions are sharing common Question word (What, why, how, when,..), then there will be a higher probability that these questions will be duplicate as compared to questions having different question words.
    •  
  • Number of common words may also play a role to decide if questions are duplicate or not.
    •  
  • Length of questions can also play a role. so there can be many possible features. We will take a few of them and analyze.

Let us now construct a few features like:  - 1. freq_qid1 = Frequency of qid1's  - 2. freq_qid2 = Frequency of qid2's  - 3. q1len = Length of q1  - 4. q2len = Length of q2  - 5. q1_n_words = Number of words in Question 1  - 6. q2_n_words = Number of words in Question 2  - 7. word_Common = (Number of common unique words in Question 1 and Question 2)  - 8. word_Total =(Total num of words in Question 1 + Total num of words in Question 2)  - 9. word_share = (word_common)/(word_Total)  - 10. freq_q1+freq_q2 = sum total of frequency of qid1 and qid2  - 11. freq_q1-freq_q2 = absolute difference of frequency of qid1 and qid2 

Here 11 new features are created using domain knowledge that can help our model to perform better

# To make model robust we have to replace a few most frequestly used words into their root words
from nltk.stem import PorterStemmer
from bs4 import BeautifulSoup
import re
def preprocess_data(data):
    data = str(data).lower()
    # To make model robust we have to replace a few most frequestly used words into their root words
    data = data.replace(",000,000", "m").replace(",000", "k")\
                           .replace("won't", "will not").replace("cannot", "can not").replace("can't", "can not")\
                           .replace("n't", " not").replace("what's", "what is").replace("it's", "it is")\
                           .replace("'ve", " have").replace("i'm", "i am").replace("'re", " are")\
                           .replace("he's", "he is").replace("she's", "she is").replace("'s", " own")\
                           .replace("%", " percent ").replace("₹", " rupee ").replace("$", " dollar ")\
                           .replace("€", " euro ").replace("'ll", " will")
    data = re.sub(r"([0-9]+)000000", r"\1m", data)
    data = re.sub(r"([0-9]+)000", r"\1k", data)
    
    
    # defining porter stemmer for stemming
    porter = PorterStemmer()
    pattern = re.compile('\W')
    
    if type(data) == type(''):
        data = re.sub(pattern, ' ', data)
    

    if type(data) == type(''):
        data = porter.stem(data)
        temp = BeautifulSoup(data)
        data = temp.get_text()
               
    
    return data

5.2 Advanced Feature Extraction (FuzzyWuzzy string matching)¶

Definition:

  • Toekn: Breaking sentence in different words on space (ex: "Arpit is a student" => ['Arpit','is','a','student']).
  • Stop_Word : list most common words, which do not have much impact on model predictions.
  • Word : List of all words which are not stopword.

Features:

  • cwc_min : Ratio of common_word_count to min lenghth of word count of Q1 and Q2
    cwc_min = common_word_count / (min(len(q1_words), len(q2_words))

  • cwc_max : Ratio of common_word_count to max lenghth of word count of Q1 and Q2
    cwc_max = common_word_count / (max(len(q1_words), len(q2_words))

  • csc_min : Ratio of common_stop_count to min lenghth of stop count of Q1 and Q2
    csc_min = common_stop_count / (min(len(q1_stops), len(q2_stops))

  • csc_max : Ratio of common_stop_count to max lenghth of stop count of Q1 and Q2
    csc_max = common_stop_count / (max(len(q1_stops), len(q2_stops))

  • ctc_min : Ratio of common_token_count to min lenghth of token count of Q1 and Q2
    ctc_min = common_token_count / (min(len(q1_tokens), len(q2_tokens))

  • ctc_max : Ratio of common_token_count to max lenghth of token count of Q1 and Q2
    ctc_max = common_token_count / (max(len(q1_tokens), len(q2_tokens))

  • last_word_eq : Check if last word of both questions is equal or not
    last_word_eq = int(q1_tokens[-1] == q2_tokens[-1])

  • first_word_eq : Check if First word of both questions is equal or not
    first_word_eq = int(q1_tokens[0] == q2_tokens[0])

  • abs_len_diff : Absolute length difference
    abs_len_diff = abs(len(q1_tokens) - len(q2_tokens))

  • mean_len : Average Token Length of both Questions
    mean_len = (len(q1_tokens) + len(q2_tokens))/2

  • longest_substr_ratio : Ratio of length longest common substring to min lenghth of token count of Q1 and Q2
    longest_substr_ratio = len(longest common substring) / (min(len(q1_tokens), len(q2_tokens))

        return token_features
    
    # Getting set stopwords in both the questions
    x_stopwords = set([k for k in x_tokens if k in STOPWORDS])
    y_stopwords = set([k for k in x_tokens if k in STOPWORDS])

    # Getting set non-stopwords in both the questions
    x_words = set([k for k in x_tokens if k not in STOPWORDS])
    y_words = set([k for k in x_tokens if k not in STOPWORDS])
    
    # Getting length of common stop words
    common_stop_count = len(x_stopwords.intersection(y_stopwords))
    
    # Getting length of common non-stop words
    common_word_count = len(x_words.intersection(y_words))
    
    # Getting common tokens from questions pairs
    common_token_count = len(set(x_tokens).intersection(set(y_tokens)))
    
    # making featues
    token_features[0] = common_word_count / (min(len(x_words), len(y_words)) + eps)
    token_features[1] = common_word_count / (max(len(y_words), len(y_words)) + eps)
    token_features[2] = common_stop_count / (min(len(x_stopwords), len(y_stopwords)) + eps)
    token_features[3] = common_stop_count / (max(len(x_stopwords), len(y_stopwords)) + eps)
    token_features[4] = common_token_count / (min(len(x_tokens), len(y_tokens)) + eps)
    token_features[5] = common_token_count / (max(len(x_tokens), len(y_tokens)) + eps)
    
     # Last word of both question is same or not
    token_features[6] = int(x_tokens[-1] == y_tokens[-1])
    
    # First word of both question is same or not
    token_features[7] = int(x_tokens[0] == y_tokens[0])
    
    token_features[8] = abs(len(x_tokens) - len(y_tokens))
    
    #Average Token Length of both Questions
    token_features[9] = (len(x_tokens) + len(y_tokens))/2
    
    return token_features

def extract_features(df):
    # preprocessing each question
    df["question1"] = df["question1"].fillna("").apply(preprocess_data)
    df["question2"] = df["question2"].fillna("").apply(preprocess_data)

    print("Adding token features")
    
    # Merging Features with dataset
    
    token_features = df.apply(lambda x: get_token_features(x["question1"], x["question2"]), axis=1)

    df["cwc_max"]       = list(map(lambda x: x[1], token_features))

    df["csc_max"]       = list(map(lambda x: x[3], token_features))
